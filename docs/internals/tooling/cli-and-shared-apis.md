# CLI modes and shared in-memory APIs

The executable entry point in [`src/main.cpp`](../../../src/main.cpp) delegates
to [`runCli`](../../../src/cli.cpp). Reusable callers should normally use
[`compileToBytecode`](../../../src/compiler/compiler.hpp) and
[`executeBytecode`](../../../src/interpreter/interpreter.hpp), the same
in-memory boundaries used by the CLI and benchmark.

## CLI flow

`parseArgs()` expands shortcuts, invokes handlers from the `options` table, and
defaults an unset mode to `CompileAndInterpret`. `validateArgs()` checks target
existence, requires `--out` for compile-only mode, and warns that debug bytecode
only affects compilation. `runCli()` returns `InvalidCli` if validation fails;
otherwise `executeCommand()` dispatches the selected path.

| Mode | Flags | Path |
|---|---|---|
| compile and run | default or `--buildAndRun` | source file -> `compileToBytecode()` -> `istringstream` -> `Interpreter::interpret()` |
| compile only | `--compile` / `-c` | `compile()` -> `writeFile()` callback; requires `--out` / `-o` |
| interpret only | `--interpret` / `-i` | bytecode file -> `Interpreter::interpret()` |

Other options are `--target`/`-t`, `--verbose`/`-e`,
`--threads`/`-h`, and `--debug`/`-d`. Note that `-h` means thread count, not
help.

## Shared APIs

`compileToBytecode(args, source, bytecode)` calls the general `compile()` with
a callback that moves generated bytecode into a string. It returns an
`ExitCode`; syntax errors are not exceptions.

`executeBytecode(args, bytecode, stats)` parses instructions, transfers their
vector to a shared store, builds a `Subprogram`, repairs each instruction's
`program` pointer, then starts an `Executor`. It returns `void` and throws typed
[`Error`](../../../src/error.hpp) for parse or execution failure. Pass an
optional [`ExecutionStats`](../../../src/interpreter/executor.hpp) to count
dynamic instruction executions.

`Interpreter::interpret()` is the logging/return-code adapter around
`executeBytecode()`. Benchmarks use the lower-level throwing API so they can add
the benchmark path to an error before reaching their application boundary.

## Invariants and change hazards

- Keep CLI, tests, and benchmarks on these shared entry points. Reassembling a
  parser/executor pipeline elsewhere will drift.
- Populate every `CliArgs` field in library callers; aggregate initialization
  otherwise leaves omitted scalars zero/false, including `threads`.
- A positive thread count is essential. CLI parsing uses `atoi` and validation
  currently does not reject zero or negative thread counts.
- Keep source and bytecode streams alive for the duration of their respective
  calls, but no backing buffer is retained afterward.
- Interpret-only currently has a defect: the `CliMode::Interpret` branch calls
  `interpreter.interpret(stream)` without returning it. After successfully
  executing bytecode it falls through and returns `InvalidCli`. Fixing this
  requires `return interpreter.interpret(stream);` plus CLI regression tests.
- `--debug` affects every path that compiles source. `validateArgs()` warns for
  compile-and-run even though that path still passes `debugBytecode` to the
  compiler and the interpreter accepts the emitted comment lines. In
  interpret-only mode the flag has no effect on an existing bytecode file.

## Observed mode example

Compile source to bytecode:

```sh
build/CLI -c -t /tmp/parallel-runtime-docs-values-20260723.ox -o /tmp/parallel-runtime-docs-values-20260723.bc -d
```

```text
[Info at compiler]: Parsed 29 tokens. Building AST...
[Info at compiler]: Built AST
[Info at compiler]: Linked graph
[Info at compiler]: Wrote bytecode to file!
# exit code: 0
```

Then interpret the bytecode:

```sh
build/CLI -i -t /tmp/parallel-runtime-docs-values-20260723.bc -h 1
```

```text
5
true
true
false
answer=42, enabled=true
# current exit code: 1 (the interpret-only fallthrough defect above)
```

Argument parsing and validation tests are in
[`tests/cli.cpp`](../../../tests/cli.cpp). Shared API integration coverage is in
[`tests/benchmark/integration.cpp`](../../../tests/benchmark/integration.cpp).
